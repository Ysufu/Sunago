Firebase vs
Supabase

Dua pilihan database gratis yang siap pakai untuk menyimpan foto kenangan pengunjung Sunago secara permanen di server sungguhan.

🔥 Firebase ⚡ Supabase
Dibuat oleh Google Open source (PostgreSQL)
Gratis selamanya? Ya, Spark plan ✓ gratis Ya, Free tier ✓ gratis
Storage foto gratis 5 GB 1 GB
Database gratis 1 GB (Firestore) 500 MB (PostgreSQL)
Realtime update ✅ Bawaan (onSnapshot) ✅ Bawaan (channel subscribe)
Kemudahan setup ⭐⭐⭐⭐ Sangat mudah ⭐⭐⭐⭐ Sangat mudah
Cocok untuk Project cepat, pakai ekosistem Google Project yang butuh SQL / lebih transparan
Lihat data via dashboard Firebase Console (UI bagus) Supabase Table Editor (seperti spreadsheet)
File kode yang diubah versi-firebase/spot-foto.html versi-supabase/spot-foto.html
🔥 Firebase — Pilih ini jika...
Mau yang paling simpel & cepat
Setup 10 menit, storage lebih besar (5GB gratis), realtime bawaan Google. Recommended untuk pemula.
⚡ Supabase — Pilih ini jika...
Mau lihat data seperti spreadsheet
Dashboard lebih intuitif, data bisa di-filter/sort/export langsung dari browser. Cocok kalau mau manage sendiri.

Opsi 1
Setup Firebase
1
Buat Project Firebase
Buka console.firebase.google.com → Login Google → Klik "Add project" → Nama bebas (misal: sunago-kopi) → Disable Google Analytics → Create project.
2
Aktifkan Firestore Database
Di sidebar kiri → Firestore Database → Create database → Pilih "Start in test mode" → Region: asia-southeast1 (Singapore) → Enable.
3
Aktifkan Storage
Di sidebar → Storage → Get started → Test mode → Next → Done. Storage untuk nyimpan file foto aslinya.
4
Daftarkan Web App & Copy Config
Di sidebar → Project Settings (ikon ⚙️) → Scroll ke bawah → Klik ikon "</>" → Register app → Copy firebaseConfig.
5
Set Security Rules Firestore
Firestore Database → Rules tab → Ganti dengan rules berikut → Publish:
firestore.rules 
// Firestore Security Rules — paste di Firebase Console
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /kenangan/{doc} {
      // Semua orang bisa baca
      allow read: if true;
      // Boleh insert kalau data valid
      allow create: if request.resource.data.keys()
                      .hasAll(['name', 'caption', 'photoUrl', 'ts'])
                    && request.resource.data.caption.size() < 500;
      // Boleh hapus (bisa diperketat nanti)
      allow delete: if true;
    }
  }
}
6
Set Security Rules Storage
Storage → Rules tab → Ganti dengan rules berikut → Publish:
storage.rules 
// Storage Security Rules — paste di Firebase Console
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /kenangan/{filename} {
      allow read: if true;
      // Hanya gambar, maks 5MB
      allow write: if request.resource.size < 5 * 1024 * 1024
                   && request.resource.contentType.matches('image/.*');
    }
  }
}
7
Paste Config ke File HTML
Buka versi-firebase/spot-foto.html → cari bagian firebaseConfig → ganti semua nilai GANTI_... dengan config yang tadi di-copy.
spot-foto.html — bagian yang diubah 
// Ganti bagian ini dengan config Firebase kamu:
const firebaseConfig = {
  apiKey:            "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX",
  authDomain:        "sunago-kopi.firebaseapp.com",
  projectId:         "sunago-kopi",
  storageBucket:     "sunago-kopi.appspot.com",
  messagingSenderId: "123456789012",
  appId:             "1:123456789012:web:abcdef123456"
};

// Ganti juga nomor WA Sunago:
const WA_NUMBER = "6281234567890"; // tanpa + dan spasi
✅ Selesai! File versi-firebase/spot-foto.html langsung jalan setelah config diisi. Upload ke hosting mana pun (Netlify, Vercel, cPanel, dll) — tidak perlu backend server.

Opsi 2
Setup Supabase
1
Buat Akun & Project
Buka supabase.com → Sign up (bisa pakai GitHub) → New project → Nama: sunago → Set database password (catat!) → Region: Southeast Asia (Singapore) → Create.
2
Buat Tabel "kenangan"
Sidebar → Table Editor → New table → Nama: kenangan → Aktifkan RLS (Row Level Security) → Tambah kolom-kolom berikut:
SQL — bisa jalankan di Supabase SQL Editor 
-- Jalankan ini di Supabase → SQL Editor → New query

CREATE TABLE kenangan (
  id           bigserial    PRIMARY KEY,
  photo_url    text         NOT NULL,
  storage_path text,
  name         text         DEFAULT 'Anonim',
  caption      text,
  ts           bigint       NOT NULL,
  created_at   timestamptz  DEFAULT now()
);

-- Aktifkan RLS
ALTER TABLE kenangan ENABLE ROW LEVEL SECURITY;

-- Policy: semua orang bisa baca & insert & hapus
CREATE POLICY "public_access" ON kenangan
  FOR ALL USING (true)
  WITH CHECK (true);
3
Buat Storage Bucket
Sidebar → Storage → New bucket → Nama: kenangan → Centang "Public bucket" → Save. Lalu klik bucket → Policies → Add policy:
SQL — Storage Policy 
-- Jalankan di SQL Editor untuk Storage policy
CREATE POLICY "public storage access"
  ON storage.objects FOR ALL
  USING ( bucket_id = 'kenangan' )
  WITH CHECK ( bucket_id = 'kenangan'
    AND octet_length(content) < 5242880 ); -- maks 5MB
4
Ambil URL & Key
Sidebar → Settings → API → Copy Project URL dan anon public key.
5
Paste ke File HTML
Buka versi-supabase/spot-foto.html → cari bagian config → ganti nilai GANTI_...:
spot-foto.html — bagian yang diubah 
// Ganti bagian ini di versi-supabase/spot-foto.html:
const SUPABASE_URL = 'https://abcdefgh.supabase.co';
const SUPABASE_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';

// Ganti juga nomor WA Sunago:
const WA_NUMBER = '6281234567890'; // tanpa + dan spasi
✅ Selesai! File versi-supabase/spot-foto.html langsung jalan. Tidak perlu backend server — Supabase SDK langsung dari browser.

Langkah Terakhir
Deploy ke Hosting

Setelah config diisi, website tinggal diupload ke hosting. Keduanya (Firebase & Supabase) bekerja di static hosting mana pun — tidak perlu server PHP atau Node.js.

🟢 Netlify (Paling Mudah — Gratis)
Buka netlify.com → Sign up
Drag & drop folder sunago ke dashboard
Otomatis dapat URL: sunago.netlify.app
Update = drag & drop lagi
🔵 Vercel (Juga Gratis)
Buka vercel.com → Sign up
Import project atau drag folder
Otomatis dapat URL: sunago.vercel.app
Custom domain bisa ditambahkan
🟠 cPanel (Hosting Berbayar)
Login cPanel → File Manager
Upload isi folder sunago ke public_html
Akses via domain yang sudah dimiliki
Cocok kalau sudah punya domain .com

sunago/

⚠️ Penting: Jangan upload kedua versi sekaligus. Pilih satu — Firebase atau Supabase — rename filenya jadi spot-foto.html lalu taruh di folder spots/.
💡 Ganti nomor WA: Cari teks 628000000000 di file HTML dan ganti dengan nomor WhatsApp Sunago yang benar. Format: 628 + nomor tanpa angka 0 di depan. Contoh: 0812-3456-7890 → 6281234567890.
